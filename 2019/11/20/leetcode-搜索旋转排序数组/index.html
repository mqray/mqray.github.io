假设按照升序排序的数组在预先未知的某个点上进行了旋转。

( 例如，数组 [0,1,2,4,5,6,7] 可能变为 [4,5,6,7,0,1,2] )。

搜索一个给定的目标值，如果数组中存在这个目标值，则返回它的索引，否则返回 -1 。

你可以假设数组中不存在重复的元素。

你的算法时间复杂度必须是 O(log n) 级别。

解题思路

题目要求算法复杂度为O(logn)，联想到是使用二分法求解。

首先，考虑到这是一个旋转数组，它有两种可能性

  1. 没有执行预先的旋转操作，也即，当前数组仍是一个递增数组
  2. 执行了预先的旋转操作，当前数组可拆分为两个递增数组A、B，整个序列中，A的最后一个数和B的第一个数不满足nums[i-1]<nums[i],左边的A偏向于大于B

问题是求解给定数target，求其下标，本质是搜索，自然而然涉及搜索–>二分。

那么，该如何进行二分操作呢？

算法过程

  1. 首先，应该找到，整个数组中的最小值及其索引值index。

    • 如果nums[0]<nums[right]，则说明这是一个未进行旋转的数组，直接返回0代表最小数下标
    • 针对旋转后的数组，在进行最小索引值的判断时，可以使用二分搜索进行查找mid = (left+right)//2，应该注意，
      • 找到nums[mid-1]>nums[mid]即找到了最小值对应的索引值，返回mid即可
      • 如果不满足上面的条件则说明nums[mid]可能落入A、B两个区间，需要进行细致搜索；
      • 如果nums[mid]>=nums[0]，说明nums[mid]同样也大于B中所有元素，最小值必然在B那么接下来需要搜索的区域就是[mid+1,right]
      • 如果nums[mid]<nums[0],则锁定最小值区间在[left,mid-1]

  2. 判断target属于index的左区间还是右区间，

  3. 然后在选定的区间中进行二分搜索。

    • 二分搜索比较容易写出
    • 针对target所落区间，进行二分搜索。
    • 如果target>=nums[0],则说明target不可能出现在B中，所以二分搜索的范围是[0,index]
    • 否则，二分搜索的范围为[index,right]

除此之外，应该考虑一些特殊情况:

  • 如果为数组为空，不进行index的查找，直接返回-1；
  • 如果数组仅有一个数，如果nums[0]==target则直接返回0；若nums[0]!=target，则返回 -1；
  • 针对index=0的情况，能直接确定二分搜索的范围为[0,right]，而不用进行复杂的判断、
