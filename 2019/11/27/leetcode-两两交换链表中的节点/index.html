leetcode_两两交换链表中的节点

发表于 2019-11-27 更新于 2019-11-28 分类于 leetcode 阅读次数： Valine：
本文字数： 1.4k 阅读时长 ≈ 1 分钟

给定一个链表，两两交换其中相邻的节点，并返回交换后的链表。

你不能只是单纯的改变节点内部的值，而是需要实际的进行节点交换。

示例：

给定 1->2->3->4, 你应该返回 2->1->4->3.

递归思路

显然，对于存在的链表节点数为0或者为1，直接返回单链表头指针。而对于长度大于2的链表，一次两两交换不能完成操作，所以需要继续进行交换。这个过程比较容易理解。递归过程中需要连接后续的递归操作，在递归版本中不需要使用phead=head保存头结点，因为程序总是递归进行，无法记录phead值。

空间复杂度为O(N/2)，时间复杂度也为O(N/2)，测试开销为：

执行用时 :24 ms, 在所有 python 提交中击败了47.76%的用户

内存消耗 :11.9 MB, 在所有 python 提交中击败了13.67%的用户

非递归实现

非递归实现中，其实思路和递归版本一致，只是递归版本中使用在递归过程中使用栈保存了最后返回的头结点tmp，而在非递归版本中，需要显示的记录初始的头结点，用于程序最后的返回。另外，在遍历规程中，需要不断移动指针以定位下次处理的节点，所以还需额外的一个指针tmp，后续操作，就是正常的节点交换过程。但是一定要注意节点交换时，谁在前，谁在后。

时间复杂度为O(N/2)，空间复杂度为O(1)，测试开销如下：

执行用时 :28 ms, 在所有 python 提交中击败了25.45%的用户

内存消耗 :11.8 MB, 在所有 python 提交中击败了24.03%的用户

体会

链表问题中，常常会用到prehead = ListNode()，方便后续处理，在这一题的非递归解法中，也不需要提前考虑head.next head.next.next是否存在，增加的虚节点很好地解决了这个问题。
