leetcode_三数之和

发表于 2019-11-12 阅读次数： Valine：
本文字数： 2.3k 阅读时长 ≈ 2 分钟

给定一个包含 n 个整数的数组 nums，判断 nums 中是否存在三个元素 a，b，c ，使得 a + b + c = 0 ？找出所有满足条件且不重复的三元组。

注意：答案中不可以包含重复的三元组。

例如, 给定数组 nums = [-1, 0, 1, 2, -1, -4]，

满足要求的三元组集合为：
[
[-1, 0, 1],
[-1, -1, 2]
]

暴力求解

暴力破解就是一层一层的分解，慢慢逼近两数之和，但是出现了超出时间限制。如下的代码，简直是令人深恶痛绝！！！！我感觉如下代码的时间复杂度是O(N^4)，因为$in$和$not in$ 的时间复杂度都是$O(N)$

排序+双指针对撞

本体的难点在于，如何消去所有存在的重复解，我尝试过，在加入res之前先对这个符合要求的list进行排序得到A，然后判断这个A是否在现存的res中，如果不存在，则加入，但是返回的总是None值。

下面说一下，首先 会用排序，会让整个程序在判别过程中很自然的只用在乎限制条件的大小，如果小了就往右移，大了就往左移；如果不使用排序的话，遍历整个无序列表的时间复杂度可能会到达O(N^3).

算法过程

  1. 首先判断这个list是否有超过三个元素，如果没有，直接返回空列表[]。
  2. 对原始list进行排序，得到从小到大排列的数字【好处是，遍历这个list时，如果nums[i]大于0，则可以直接返回结果】
  3. 对于排序后的列表，如果连续两个值是一致的，我们应该只用考虑它一次【这里是指在第一层循环中的nums[i]】
  4. 设置左右指针，起始地址应该是i+1，len(nums)-1
  5. 判断这三个指针指向的内容的和Limit与限制条件之间的关系，如果大于限制条件，则说明，right指针应该指向一个稍微小一点的数，即right-1；而小于限制条件时，应该将left指向一个更大的数，即left+1
  6. 而当找到符合条件的三元组时，先将它加入res中，
  7. 除此之外，应该判断left之后，right前 是否具有相同的值，如果有，应该排除。这里和第3步中的目的一样，但是不同的是，第3步中是第一层循环，在计算和之前提前判断，而这里是内层循环判断是否存在重复的解；
  8. 而当排除了这些重复解之后，应该确保能检索到这一层（i）循环里所存在的所有解，是为了算法的继续进行。没有这两句，程序会一直死循环

这里举个例子吧

nums = [-4,-4,0,0,1,1,2,2,3,3,4,4]

第三步是为了不重复考虑-4与其他数字的组合

而当确定nums[i]=-4之后，能找到[-4,0,4]这组解，而第7步是为了不重复计算两个0，以及两个4产生的结果。

复杂度分析

时间开销在于 对列表的先排序以及遍历列表以及指针对撞两部分，对应的：

时间复杂度：排序：O(NlogN) + 第一层遍历列表O(N)*第二层指针对撞遍历O(N) = O(N^2)

空间开销在于用来存储最终结果的列表

空间复杂度：O(1)

另外，哪怕算法思想都是一样，但是提交结果得到的时空开销会有差异，而有些简单的判别语句就会很明显的减少循环次数，如第一层循环时，如果nums[i]大于0，就不用继续求解了。加上只用424ms，而少了则要花508ms。这就是可调优的部分。
