这里可以延伸为查找两个数组中的第k小的数，由于题目中明确给出了时间复杂度的限制条件，很自然的可以联想到二分法；而此处的二分法是如何进行的呢？

  • 首先，对于数组A和B，数组长短分别为m和n，在寻找中位数时，对于m+n为偶数，求中位数时，需要考虑合并数组中的两个数；而当m+n为奇数时，仅需考虑数组中间的数。兼顾两种情况，可以做如下操作，left = (m+n+1)//2；right = (m+n+2)//2；当m+n为偶数时，left和right是大小不同的两个值，而是奇数时，它们能取到同一个数。

  • 其次，K确定后，对数组A、B查找编号为k//2的数，判断大小，数值小的数组，前K//2个数可以排除，后续的操作中，更新这个K值，只用继续判断A[K//2:]和B即可，后续的处理是可以递归执行的。

  • 那么，递归的出口是什么？

  • 很明显的K等于1时，只用判断此时，哪个数是数组中的最小值即可
