组合总数 I

给定一个无重复元素的数组 candidates 和一个目标数 target ，找出 candidates 中所有可以使数字和为 target 的组合。

candidates 中的数字可以无限制重复被选取。

说明：

  • 所有数字（包括 target）都是正整数。

  • 解集不能包含重复的组合。

示例 1:

输入: candidates = [2,3,6,7], target = 7,
所求解集为:
[
[7],
[2,2,3]
]

示例 2:

输入: candidates = [2,3,5], target = 8,
所求解集为:
[
[2,2,2,2],
[2,3,3],
[3,5]
]

回溯法

拿到之后基本没什么思路，后来看了题解，发现牛人是真的多，是真的厉害。

解法是怎样的的呢？思想是将target利用nums中的数字进行拆解，拆解的条件是拆解后得到的值如果等于0那么停止拆解，如果拆解后的值小于0，与题意不符，停止拆解。每一次拆解都相当与在以target为根节点的树上开枝散叶，整个过程递归调用拆解过程，最后形成一个深度优先遍历树。

另外对于可能出现的重复解，这里的做法是，现将原始数组排序，然后，对于已经使用过的拆解字，不能再继续使用【题目中说无重复数】

算法分析

本题与组合总和I的差异在于，候选数组candidates中可能出现重复数字，另外对于每个数字，只能出现一次【相同的值其位置不同算不同数，可以一起出现】，上一题中为了避免重复解，在剪枝的过程中使用的是避免后续子树重复度量已经在前面左子树中计算过的组合，所以传入的是candidates[i:]，值得注意的是，在做最左侧的左子树是有可能出现重复使用最小值的【比如数组的第一个参数是1，在深度遍历过程中它会频繁使用】，而在这里对于每一个数字，他只能出现一次。所以在递归调用时，需要传入一个标识当前遍历位置的参数，以免数字的重复使用。DFS(candidates,target-candidates[i],i+1,tmp+[candidates[i]])的作用和之前的组合总和I的目的一样，只是两题的问题限定稍有区别。
