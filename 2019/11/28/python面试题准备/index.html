元类

什么是元类？元类的应用？

通常，我们定义类来创建对象，但是现在我们知道类也是对象。那么是通过什么来创建类呢？答案就是元类。

可以使用type(类名,基类元组(可以为空，用于继承), 包含属性或函数的字典)动态创建类对象，并且能够给类添加更多的方法和属性。*type是内置的元类，Python默认使用它来创建类 *。

也可以定义属于自己的元类。 创建类的时候，可以给它添加etaclass属性。

99%的开发者不会用到元类，元类主要用于创建API,如Django中的ORM

类属性、实例属性的区别

类属性归类所有，所有实例都能访问；而实例属性则是实例独有。

类方法、实例方法、静态方法的区别

类方法

使用装饰器@classmethod，第一个参数必须是当前类对象，参数名约定为cls，通过它来传递类的方法和属性，而不能传递实例的属性和方法。

类对象和实例对象都可以调用。

静态方法

使用装饰器@staticmethod，不需要传入self和cls，可以通过类名和对象名调用，但不能访问实例属性。

不能访问类变量和实例变量。

实例方法【一般方法】

第一个参数必须是实例对象，约定为self，用来传递实例的方法和属性【也可以传递类的属性和方法】

但只能由实例对象调用。

遍历一个object的所有属性，并打印每一个属性名？

可以使用两个方法：obj.__dict__(),dir(obj)

前者一般是字典或其他映射对象，用来存储一个对象的属性。但是内建类型对象不存在这个属性；并且它只返回当基类【这里是基类而不是父类因为可以通过接口实现多重继承】的属性字典，并不会返回其父类的属性。

而dir(obj)会返回当前类以及它的所有基类的类属性名，即当前类及所有基类的__dict__键值。

python解释器介绍？

  • CPython，官方版本的解释器，它是用C语言开发的，所以叫CPython。
  • IPython，基于CPython的一个交互式教师器，相较CPython其交互方式有所增强。
  • PyPy，其目标是执行速度，采用JIT技术，对python代码进行动态编译，可以显著提高python代码的执行速度。
    • JIT编译，即Just-in-time compilation，指当某段代码即将第一次被执行时进行编译，因而叫即时编译，是动态编译的特例。
    • 动态编译，即dynamic compilation，指在运行时进行编译
    • 事前编译，即ahead-of-time compilation，AOT，也叫做静态编译static compilation
    • 自适应动态编译，即adaptive dynamic compilation，通常执行的时机比JIT迟，先让程序以某种形式运行起来，收集某些信息后再做动态编译。
  • Jpython，运行在JAVA平台下的pyhon解释器，可以直接把python代码编译为JAVA字节码执行。
  • IronPython，运行在微软.NET平台下的解释器，可以直接将python代码编译为.NET字节码执行。

解释型语言和编译型语言

计算机不能理解高级语言，更不能执行高级语言，任何使用高级语言编写的程序若要在计算机上执行，需要先转化为机器语言，也即机器码。转化的方式有两种：

  • 编译 代表有C、C++、Objective C，是指针对特定平台。使用专门的编译器，将高级程序语言的源代码一次性的编译为可悲该平台执行的机器码，并包装为该平台所能识别的可执行程序的格式。
    • 优点是：源代码编译为机器语言的文件，再运行时，直接使用编译结果即可，运行时无需重新编译，所以编译型语言的执行效率更高
  • 解释 代表有python、java，解释型语言不需要事先编译，直接将源代码解释成机器码执行，故而只需要平台提供相对应的解释器即可运行该程序。
    • 每次运行都需要将袁大妈解释为机器码，效率低；
    • 平台提供对应的解释器，即可运行源代码，可移植性高；

python代码被编译为字节码【类似汇编指令的中间语言】；由python虚拟机执行编译过后的字节码

一条python语句对应若干个字节码指令，每一个字节指令又对应着一个函数偏移量，可视作指令ID

虚拟机一条一条执行字节码指令，进而完成程序执行。【dis模块可以对CPython编译产生的字节码进行反汇编，生成字节码指令】

map()和reduce()函数

map()函数，格式如下：map(func,seq1[,seq2...])

其作用是将func作用于seq的所有元素，并返回一个map对象

  • 如果func为None，作用同zip()

  • 当只有一个seq时，其效果等同于列表推导式

  • 当有多个seq时，map()能够并行处理多个seq【此时函数就需要传入多个参数】，但是要保证这些seq的长度一致。否则会抛出类型异常。

reduce()函数，格式如下：reduce(func,seq[,int])

  • zip()函数将多个可迭代对象Iterable组合为一个元组tuple序列的迭代器Iterator。
  • zip(*)是zip()的逆过程。

作用是化简，将上次的迭代结果与下一个元素一同执行一个二元func函数。参考 Python函数式编程——map()、reduce()

  • 如果指定int值，则第一次迭代时使用该元素；如不指定，则使用seq的第一个元素
  • 使用reduce()实现阶乘

什么是hash？

哈希表（Hash table，也叫散列表），是根据关键码值(Key value)而直接进行访问的数据结构。也就是说，它通过把关键码值映射到表中一个位置来访问记录，以加快查找的速度。这个映射函数叫做散列函数，存放记录的数组叫做散列表。

python在散列的过程中，会根据当前表的疏松程度进行空间的再申请。当使用量超过总槽数的2/3，就会申请不少于当前活动槽数4倍的新空间

可参考HashTable和HashMap的区别详解、【Java集合源码剖析】HashMap源码剖析、Hashing

python中的可变对象和不可变对象？

抽象数据类型的操作一般分为三类：

  1. 构造操作: 这类操作主要是基于一些已知信息，产生这个类的实例对象。类似银行进行开通账户

  2. 解析操作：这类操作是获取对象的一些有用信息，其结果反应了对象的一些特征，但返回的不是对象本身。类似银行查询账户余额

  3. 变化操作 ：这类操作是修改对象内部的信息和状态。比如一个银行账户进行转账操作

如果一个类型，具只具有1和2两种操作，也就说只具有构造和解析操作，那么这个类型就是不可变类型，这个类型的对象就是不可变对象
如果一个类型，具有1、2、3三种操作，这个类型就是可变类型，这个类型的对象就是可变对象。

不可变对象

对象所指向的内存中的值不能被改变，当改变这个变量的时候，原来指向的内存中的值不变，变量不再指向原来的值，而是开辟一块新的内存，变量指向新的内存。

python中的不可变对象有数字int、float，字符串str，元组tuple以及布尔类型boolean

但是要注意整数中的最小整数池【-6,255】

小整数对象池

整数在程序中的使用非常广泛，Python为了优化速度，使用了小整数对象池， 避免为整数频繁申请和销毁内存空间。

Python 对小整数的定义是 [-5, 257) 这些整数对象是提前建立好的，不会被垃圾回收。在一个 Python 的程序中，无论这个整数处于LEGB中的哪个位置，

可变对象

对象指向的内存中的值会改变，当更改这个变量的时候，还是指向原来内存中的值，并且在原来的内存值进行原地修改，并没有开辟新的内存。

python中的可变对象有集合set，字典dict，列表list

字典的键值中的key必须是不可变类型，如数字，字符串，元组，而列表、元组、字典是可变类型

python中的魔术方法

python中以__双下划线包起来的方法均称作魔法方法。主要包含以下几类：

  • 构造和初始化 __init__(),__new__(),两者共同构成构造函数，__del__()，后者可以理解为构析函数
  • 控住属性访问 __getattr__(self,name),__setattr__(self,name,val),__delattr__(),__getattribute__(self,name)
  • 创建自定义容器__getitem__,__setitem__,__delitem__等
  • 用于比较、数值计算、类型转化的方法
  • 复制对象__copy__
  • 调用对象__call__

谈谈你对面向对象的理解

首先针对python，我们常说python万物皆对象，为什么呢？在python环境中输入没有声明的变量a，报错信息是NameError: name 'a' is not defined，而不是Variable，因为在python中将所有的函数名、变量名，类型名等标识符都称作名字，在名字空间中总有一个对象与之相关联，所以说万物皆对象。

python中的对象的核心就是一个结构体PyObject，并基于此，定义了两种类型的变量，定长变量【如float】和变长变量，后者在PyObject的基础上封装定义为PyVarObject

针对面向对象，有

  • 面向对象是基于万物皆对象这个哲学观点. 把一个对象抽象成类,具体上就是把一个对象的静态特征和动态特征抽象成属性和方法,也就是把一类事物的算法和数据结构封装在一个类之中,程序就是多个对象和互相之间的通信组成的
  • 面向对象三大特征：封装、继承、多态
  • 封装，将一类事物的属性和行为抽象为一个类，使其属性私有化，行为公开化，提高了数据隐秘性的同时，代码模块化。使得代码的复用性更高
  • 继承：将一类事物的共有属性和行为抽象为一个高层类，称作父类，每一个子类继承父类方法、属性的同时，可以扩展自己的特有属性和方法。
  • 多态：封装继承实现了代码复用，多态则实现了接口的重用，子类可以对父类中的属性或方法重写，降低了父类和子类之间的耦合程度。

设计模式

对设计模式的理解，简述你了解的设计模式

设计模式Design Pattern: 是一套被反复使用、多数人知晓的、经过分类编目的、代码设计经验的总结。使用设计模式是为了可重用代码、让代码更容易被他人理解、保证代码可靠性。

设计模式六大原则
  1. 开放封闭原则 Open Close Principle ： 对扩展开放，对修改关闭
  2. 里氏替换原则 Liskov Substitution Principle,LSP； 任何基类可以出现的地方，子类一定可以出现 。 LSP是继承复用的基石，只有当衍生类可以替换掉基类，软件单位的功能不受到影响时，基类才能真正被复用，而衍生类也能够在基类的基础上增加新的行为 。
  3. 依赖倒转原则 Dependence Inversion Principle： 是对接口编程，依赖于抽象而不依赖于具体。
  4. 接口隔离原则 Interface Segregation Principle： 使用多个隔离的接口，比使用单个接口要好。 【 降低类之间的耦合度 】
  5. 迪米特法则 Demeter Principle ： 又称最少知道原则， 一个实体应当尽量少的与其他实体之间发生相互作用，使得系统功能模块相对独立 。
  6. 合成复用原则 Composite Reuse Principle ： 尽量使用合成/聚合的方式，而不是使用继承

在《设计模式》一书中共有23中设计模式，分作创建型、结构型和行为三大类。

在对象的结构和对象的创建问题都解决了之后，就剩下对象的行为问题了，如果对象的行为设计的好，那么对象的行为就会更清晰，它们之间的协作效率就会提高
