下面准备讲讲面试中常见的排序算法，总结性的陈词，后面再补充。

先从堆排序说起。

交换排序之快速排序

快速排序的时间复杂度为O(NlogN),递归过程构建的树占用O(logN)，每一次对数组进行排序消耗O(N)。另外，快速排序不稳定。最坏情况是数组已经排好序使得每次划分点只能分开一个数，时间复杂度为O(N^2)

排序思想

交换排序的思想是，选定一个pivot，排序过后使得数组中pivot两侧的数左边小于右边。然后将这个问题继续深入的转化为更小的问题求解。具体的选定pivot是为了分割排序数组，然后选取left和right两个指针，进行如下操作：注意这里的Pivot是可以任意指定的

  • 在left<=right的前提下

  • 移动left指针找到比nums[pivot]要大的数A，停下

  • 移动right指针找到比nums[pivot]要小的数B，停下

  • 如果left<=right，则交换A、B两值，并有left+1，right-1

  • 对当前数组完成排序后，返回分隔点的索引值

选择排序之堆排序

堆排序的时间复杂度为NlogN，其中前面的logN代表建立大根堆/小根堆的时间复杂度，N代表对已建立的堆进行遍历交换排序时的时间复杂度。另外，堆排序是不稳定的。

之所以叫做堆排序，是因为在排序过程中设计到建立堆这种数据结构，所以称作堆排序。

主要需要掌握以下三点：

堆化

什么是堆化，也即heapify()，堆化的过程能使一个完全二叉树满足父节点的值大于或小于子节点，前者称作大根堆，后者称作小根堆。堆化过程中的注意事项有：

  • 交换父节点和左右子节点，使其满足大小根堆的特点。
  • 判断是否交换时，需要在左右子节点存在的条件下。
  • 堆化过程应该是递归的，从堆顶【完全二叉树树根】一直往下进行
  • 堆化的点数组下标大于等于数组长度时，退出
建立堆

建堆的过程实际上是对堆化操作的调用，但是要注意的是，建堆时，对最后一个节点的父节点及其之前节点进行堆化即可建立大根堆或小根堆。所以建堆过程是从后向前的。

排序过程

由于前面进行了建堆操作，所以此时的数据已经满足根节点大于其左右节点值【大根堆】，此时，将这个最大值与数组末尾值【也就是二叉树的最后一个节点】进行交换，此时将最大值存入数组末尾了。然后对于更新过的根节点元素，进行堆化操作，使得这一次的堆顶元素是除了已经移到数组末尾元素外的最大值，此时将它与倒数第二个叶节点进行交换，相当于将值存入数组倒数第二个位置。相当于是利用堆化过程，不断将当前的最大值放入堆顶，进而存入数组末尾。

归并排序

归并排序的时间复杂度是O(NlogN)，空间复杂度为O(N)，这里的logN是指递归过程中生成的二叉树的深度，而具有N个节点的完全二叉树的深度为O(logN)，而N则代表合并两个排序好的数组需要的开销。

归并排序的思想是对两个已经排好序的数组进行合并，故称归并排序。但问题是常见场景是数组常常是无序的，那么是否可以使用归并排序求解呢？当然是可以的，但是需要利用动态规划的思想，现在不是要求两个已经排序好的数组嘛？可以呀，我现在将这个较大问题简化，递归的完成得到需要的两个排序数组。
